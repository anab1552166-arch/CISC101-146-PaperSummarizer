# Module 4: Rendering & refinement

### Purpose
- Compose final outputs: Paper Summary, Section Table, Expert/Lay Summaries, Glossary, Checks.

### Final summary
- ≤300 words.
- Logical order: problem → method → results → interpretation → limitations.

### Formatting
- Clear headings.
- Tables for section claims/methods/terms.

### Expert vs Lay
- Expert: technical, dense.
- Lay: plain language, accessible.

### Pipeline
1. Section-by-Section Table.
2. Section summaries.
3. Paper Summary.
4. Expert + Lay Summaries.
5. Mini-Glossary (≤10 terms).
6. Checks & Warnings.

### Example skeleton
- Paper Summary
- Section-by-Section Table
- Expert Summary
- Lay Summary
- Mini-Glossary
- Checks & Warnings
